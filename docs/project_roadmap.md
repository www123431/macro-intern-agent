# Macro Alpha Pro — 项目评价与长远开发指引

> 写于 2026-04-15。基于当前架构的客观评估，以顶级量化金融分析视角 + 严谨学术标准撰写。所有结论标注边界与局限。

---

## 一、项目定位的重新确认

**Macro Alpha Pro 是什么**：系统化宏观研究辅助框架。它规范分析师的判断过程，强制结构化推理，并通过 Alpha Memory 积累可审计的决策记录。

**Macro Alpha Pro 不是什么**：Alpha engine。它不产生可直接交易的信号，不具备统计意义上经过验证的预测优势（截至本文写作时，Clean Zone 样本量不足以支撑任何实证主张）。

唯一有效的绩效证据：**Clean Zone（2025-04-01 起）的实盘决策胜率**，且需积累至统计显著水平方可引用。

---

## 二、学术意义评估

### 2.1 成立的部分

**① 对抗性推理在金融分析中的方法论应用**

蓝队/红队/仲裁三层辩论结构，有学术文献支撑。Du et al. (2023) *"Improving Factuality and Reasoning in Language Models through Multiagent Debate"* 证明多智能体辩论可显著提升 LLM 推理质量。本项目是该思路在宏观板块研究领域的具体实例化，在此垂直方向目前无公开同类工作，具备方法论原创性。

**② 认识论诚实的评估方法论**

拒绝历史回测、仅使用实盘 Clean Zone 记录评估，直接对应学术界对量化金融"复现危机"的批判：

- Harvey, Liu & Zhu (2016): *"…and the Cross-Section of Expected Returns"* — 历史回测中大量 alpha 因子无法在样本外复现
- Lopez de Prado (2018): *Advances in Financial Machine Learning* — 系统性论述回测过拟合的机制

本项目的设计决策本身即可作为方法论论点写入论文。

**③ XAI 归因 + 结构化决策记忆**

将每次判断的置信度分解（macro/news/technical confidence）、失效条件、驱动因子持久化，并作为后续分析的上下文——这是 LLM 在专业知识管理领域的具体应用，区别于通用 RAG，接近"机构经验压缩"的思路。

### 2.2 当前不成立的部分（必须承认的局限）

| 局限 | 说明 |
|------|------|
| 样本量 | Clean Zone 样本接近零，任何关于系统有效性的主张均无统计基础 |
| 非真实对抗 | 蓝队/红队使用同一模型，"分歧"源于 prompt 角色扮演而非真实观点分布差异 |
| 胜率定义不严格 | horizon 判断标准、"正确"的认定方式尚未形式化 |
| 无基准对比 | 未与简单动量因子、等权组合等 baseline 对比，无法判断系统价值来自 AI 还是来自量化因子本身 |
| LLM 不可复现性 | 相同 prompt 输出不固定，难以进行严格的消融实验 |

**当前可支撑的学术定位**：Early-stage methodological contribution（方法论框架论文），而非实证论文。

## 三、实践意义评估

### 3.1 成立的价值

- **判断过程标准化**：强制每次分析经过相同框架，减少分析师锚定偏差与叙事偏见
- **可审计的决策链**：XAI 归因在合规场景和投委会汇报中有真实使用价值
- **研究覆盖广度**：同时跟踪 18 个全球板块，小型机构/家族办公室难以用人工实现
- **速度**：新闻 + 量化数据的整合比人工研究快 10x 以上

### 3.2 边界

- 系统不替代基本面判断，只是结构化基本面判断的过程
- 输出是研究参考，不是交易指令
- 系统在信息混乱、叙事分歧大的市场环境（高 VIX）中理论上最有价值；在趋势清晰的市场里价值边际递减

## 四、当前架构的核心 Limitation

### L1. 信息优势缺失
LLM 训练于公开数据，无法接触私有信息流。所有"分析"本质是公开信息的重组。在有效市场中，这不产生信息优势。可缓解但无法根治。

### L2. 同源模型辩论
蓝队/红队使用 Gemini 同一模型。系统性偏见无法通过内部辩论被发现——两方会在相同条件下犯相同错误。

### L3. 统计验证路径漫长
假设每周产生 3 个有效决策，验证 55% 胜率达 95% 置信度需要约 400 个样本，即 2.5 年。这是客观约束，无法通过工程手段绕过。

### L4. LLM 输出不稳定
模型版本迭代（Gemini 2.5 → 3.x）会改变输出分布，历史决策和未来决策的可比性存疑。需要 prompt 版本管理和输出校准机制。

### L5. γ/λ 框架尚未闭环
目标波动率 σ_target 已可写入数据库，但 γ 的实际推导和 λ 的稳定性准则均处于 stub 状态，量化层与 AI 层之间的接口尚未打通。

## 五、长远开发指引

> 按优先级排列。高优先级 = 对 Clean Zone 有效性直接影响；低优先级 = 架构增强。

### P0：统计框架严格化（最高优先级）

**目标**：让 Clean Zone 数据有能力回答"系统是否有效"这个问题。

- 形式化"胜率"定义：明确 horizon 到期判断规则（价格变动 vs. 相对强弱 vs. 主观验证）
- 引入 Brier Score 作为概率校准指标，检验 overall_confidence 是否与实际准确率一致
- 设定 baseline：同期简单动量因子（过去 3M 动量）的命中率，作为系统价值的参照点
- 实现 binomial test 自动计算，当 n≥30 时在 Clean Zone 仪表盘显示置信区间

**技术成本**：中等。主要是 memory.py / admin.py 的统计计算逻辑。

---

### P1：多模型真实辩论（架构级改进）

**目标**：解决 L2（同源模型偏见）。

引入第二个模型作为红队（Claude Sonnet / GPT-4o），蓝队保留 Gemini。不同模型的训练数据分布和 RLHF 偏好不同，辩论结果更接近真实的观点碰撞。

**技术方案**：
- debate.py 的 `_pool_call` 拆分为 `_blue_call`（Gemini）和 `_red_call`（Anthropic/OpenAI）
- 需要独立的 key pool 或 API 接口
- 仲裁者可继续使用 Gemini 或引入第三方模型

**技术成本**：中等。主要是 API 接入和 prompt 适配。  
**注意**：引入前先积累足够 Clean Zone 数据，有对照组才能判断多模型是否真的改善判断质量。

---

### P2：Prompt 版本管理

**目标**：解决 L4（模型迭代导致输出分布漂移）。

- 每个 AI 调用记录使用的 model_version + prompt_hash
- 存入 decisions 表
- 当 Gemini 模型升级时，可以检测历史决策是否在相同 prompt 下产生不同输出分布

**技术方案**：在 `save_decision()` 中增加 `model_version` 和 `prompt_version` 字段。

**技术成本**：低。

---

### P3：γ/λ 量化框架闭环

**目标**：解决 L5，让量化层真正影响分析框架而非仅作展示。

**γ（风险厌恶）**：
- 当前：σ_target 已可写入 DB，γ 未推导
- 目标：`γ = (μ_portfolio) / (σ_target²)` 基于历史 Clean Zone 决策隐含收益率推导
- 门控：需 ≥15 个 Clean Zone 有效决策才启用，否则用先验

**λ（LASSO 正则化）**：
- 当前：stub，返回先验值 0.01
- 目标：稳定性准则——在不同 bootstrap 样本下，选出在 ≥80% 重采样中保持非零的信号
- 门控：≥15 个 Clean Zone 样本（已实现门控逻辑，未实现计算逻辑）

---

### P4：宏观 Regime 检测增强

**目标**：让系统能识别当前处于哪个宏观环境，从而调整分析权重。

当前 regime 分类（基于 VIX 阈值）过于粗糙。可引入：
- **隐马尔可夫模型（HMM）**：2-3 个隐状态，基于 VIX、信用利差、收益率曲线斜率联合识别
- 状态标签：risk-on / risk-off / transition
- 用于调整 XAI attribution 中各信号的权重

**引入条件**：先质疑必要性。当前 VIX 阈值分类是否已经足够？需要 Clean Zone 数据验证 regime 标签与判断质量的相关性，再决定是否引入 HMM。

**技术成本**：中等。hmmlearn 库，但需要参数估计的历史数据。

---

### P5：新闻质量评估层

**目标**：当前新闻注入没有可信度过滤，所有来源权重相同。

- 来源可信度评分（主流财经媒体 > 博客 > 社交媒体）
- 去重：相同事件的多篇报道合并为一条
- 时效性衰减：超过 48 小时的新闻降权

**技术成本**：低至中等。主要是 news.py 的预处理逻辑。

---

### P6：另类数据接入（长期，谨慎评估）

> 仅在 P0-P2 完成后考虑。否则无法判断另类数据是否真正改善判断。

可接入的数据类型（按成本/价值排序）：
- **期权隐含波动率曲面**（免费：CBOE 数据）：识别市场对尾部风险的定价
- **资金流数据**（付费：ETF.com 或 Bloomberg）：机构资金进出
- **政策文本分析**（免费：Fed statements, FOMC minutes）：央行立场变化

---

## 六、不建议引入的技术

| 技术 | 原因 |
|------|------|
| 强化学习（RL for portfolio） | 需要稳定的 reward signal，当前样本量完全不支持；容易过拟合 |
| 深度学习价格预测模型 | 与项目定位（研究辅助）冲突；会引入回测过拟合风险 |
| 实盘交易接口 | 超出项目边界，引入执行风险和合规风险 |
| 向量数据库（Pinecone/Weaviate） | 当前 Alpha Memory 规模下 SQLite 完全够用，过度工程 |

## 七、阶段性里程碑

| 里程碑 | 条件 | 意义 |
|--------|------|------|
| M1 | Clean Zone n≥30，胜率 >50% 且 p<0.1 | 系统有信号的初步证据 |
| M2 | Clean Zone n≥100，Brier Score < 0.25，胜率 >55% 且 p<0.05 | 可以声称系统有边际价值 |
| M3 | Clean Zone n≥400，跨越 ≥2 个宏观 regime，alpha vs. 动量 baseline 显著 | 可以讨论系统性价值 |
| M4 | 多模型辩论引入，γ/λ 框架闭环 | 架构完整，具备研究论文发表条件 |

当前位置：**M1 之前**。

---

## 八、一句话总结

系统现阶段的核心任务只有一个：**积累高质量的 Clean Zone 决策记录，并用严格的统计框架验证其有效性**。其他一切改进都应以此为前提——没有数据，架构再精妙也是空中楼阁。
